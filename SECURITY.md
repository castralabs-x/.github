# Security Policy
The following security policy applies to this organization (```castralabs-x```) and related projects.

## Reporting a Vulnerability

At X Labs and Castra Labs, we take security seriously. If you believe you have discovered a security vulnerability in any repository under the X Labs initiative, we ask that you follow responsible disclosure practices and report it to us privately before making any information public.

**Please do not open a public GitHub issue for security vulnerabilities.**

### How to Report

Send your report via email to:

📧 **security@castralabs.com**

Include as much of the following as possible to help us triage and respond quickly:

- A description of the vulnerability and its potential impact
- The affected repository, component, or dependency
- Steps to reproduce or a proof-of-concept (if available)
- Any suggested mitigations or patches

---

## Response Timeline

We are committed to responding to all security reports in a timely manner. You can expect:

| Stage | Target Timeframe |
|---|---|
| Initial acknowledgment | Within 2 business days |
| Triage & severity assessment | Within 5 business days |
| Status update | Within 10 business days |
| Resolution or mitigation | Dependent on severity and complexity |

We will keep you informed throughout the process and notify you when the issue has been resolved.

---

## Scope

This policy applies to all repositories published under the X Labs GitHub organization and any associated Castra Labs products incubated here.

**In scope:**
- Source code in this repository and related X Labs repos
- APIs and services deployed from this codebase
- Authentication and authorization logic
- Data handling and storage mechanisms

**Out of scope:**
- Third-party dependencies (please report these to their respective maintainers)
- Social engineering or phishing attacks
- Issues in infrastructure not managed by Castra Labs
- Findings from automated scanners without a demonstrated impact

---

## Disclosure Policy

X Labs follows a **coordinated disclosure** model. We ask that you:

1. Report the vulnerability to us privately and allow reasonable time to respond
2. Avoid accessing, modifying, or exfiltrating data beyond what is necessary to demonstrate the issue
3. Refrain from publicly disclosing the vulnerability until we have issued a fix or reached a mutual agreement on disclosure timing

In return, we commit to:

- Acknowledging your report promptly
- Keeping you informed of our progress
- Crediting you in any public disclosure, if you wish

---

## Supported Versions

Security fixes will be applied to the most recent stable release. Older versions may not receive backported patches unless the severity warrants it.

| Version | Supported |
|---|---|
| Latest release | ✅ |
| Previous release | ⚠️ Critical fixes only |
| Older versions | ❌ |
